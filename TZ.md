# Техническое задание на разработку системы аудита криптостойкости учетных записей

## 1. Цель проекта
Разработка специализированного веб-сервиса для специалистов в области информационной безопасности (ИБ-аудиторов). Система решает задачу наглядной демонстрации уязвимости паролей путем математического расчета энтропии, моделирования атак на распределенных вычислительных мощностях (GPU-кластерах) и проверки по глобальным базам утечек. Интеграция с искусственным интеллектом позволяет предоставлять экспертные рекомендации по укреплению безопасности.

## 2. Целевая аудитория и роли
* **Целевая аудитория:** Системные администраторы, аналитики SOC-центров, специалисты по аудиту цифровой гигиены.
* **Роли пользователей:**
    * **Гость:** ознакомительный доступ к калькулятору.
    * **Аудитор:** авторизованный пользователь с доступом к формированию кластеров, истории проверок и ИИ-аналитике.
    * **Администратор:** управление базой данных оборудования и мониторинг системных логов.

## 3. Схема данных (Сущности)
В системе реализованы три взаимосвязанные модели:

* **TargetGroup (Группа цели):** позволяет группировать проверки по категориям инфраструктуры (например: Серверный сегмент, Клиентские данные, Административные учетные записи).
* **HardwareRig (Оборудование):** справочник технических характеристик графических ускорителей, включающий показатели энергопотребления и скорости перебора хэшей (MD5/SHA-256).
* **PasswordAuditLog (Журнал аудита):** центральная сущность, связывающая результаты анализа, конкретную цель и состав использованного вычислительного кластера (связь Many-to-Many).

## 4. Ключевой функционал (User Stories)
* **Конструктор вычислительного кластера:** пользователь может формировать произвольный набор видеокарт в пуле, указывая индивидуальное количество для каждой модели. Система суммирует производительность в режиме реального времени.
* **Интеллектуальный поиск оборудования:** интеграция с Minerstat API v2.0 позволяет находить актуальные спецификации GPU по названию с автоматическим заполнением данных.
* **Анализ по протоколу k-Anonymity:** проверка пароля на наличие в утечках без передачи исходных данных во внешнюю сеть (локальное SHA-1 хэширование).
* **Экспертное сопровождение (ИИ):** генерация динамических справок об информационной безопасности и персонализированных советов по улучшению парольной политики через GigaChat API.

## 5. Внешние интеграции и технологии
* **Внешние API:**
    * GigaChat API (SberDevices): генерация контента и рекомендаций.
    * Minerstat API: получение спецификаций оборудования.
    * Have I Been Pwned API: проверка скомпрометированных данных.
* **Аналитический блок:**
    * Использование математических алгоритмов расчета энтропии Шеннона.
    * Визуализация данных с помощью Chart.js (линейный прогноз сложности и круговая диаграмма распределения мощностей).
    * Система локального кэширования внешних запросов для обеспечения отказоустойчивости.
