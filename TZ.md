# Техническое задание: Система комплексного аудита криптостойкости паролей

## 1. Цель проекта
Разработка профессионального веб-сервиса для специалистов по информационной безопасности. Система предназначена для наглядной демонстрации уязвимостей паролей путем математического расчета их энтропии и моделирования атаки методом полного перебора (Brute-force). Сервис позволяет объединять вычислительные мощности нескольких графических процессоров (GPU) в единый пул и безопасно проверять пароли по глобальным базам утечек (метод k-Anonymity).

## 2. Роли пользователей
* **Гость:** Доступ к калькулятору на главной странице без сохранения результатов.
* **Авторизованный пользователь (Аудитор):** Доступ к созданию групп целей, управлению справочником оборудования и просмотру детальной аналитики пулов.
* **Администратор:** Полный доступ к управлению БД и конфигурациям API.

## 3. Модели данных (Сущности)
В системе реализованы 3 взаимосвязанные кастомные модели:

* **Модель 1: TargetGroup (Группа целей)**
  * Описание: Классификация объектов аудита (например, "Финансы", "Персональные данные").
  * Поля: name (строка), created_at (дата/время).

* **Модель 2: HardwareRig (Оборудование)**
  * Описание: Технические характеристики GPU.
  * Поля: name (модель), power_watts (потребление), hashrate_md5 (H/s), hashrate_sha256 (H/s).

* **Модель 3: PasswordAuditLog (Журнал аудита)**
  * Описание: Фиксация результатов проведенных проверок.
  * Поля: target (ForeignKey к TargetGroup), hardware (ManyToManyField к HardwareRig для поддержки пулинга), password_length (целое), entropy_score (float), time_to_crack_seconds (float), is_pwned (boolean), tested_at (дата/время).

## 4. Ключевой функционал
* **Интеллектуальный конфигуратор оборудования:** Живой поиск видеокарт через Minerstat API v2.0 с автоматическим парсингом технических характеристик и хэшрейта.
* **Вычислительный пулинг (Pooling):** Возможность выбора нескольких устройств одновременно. Система суммирует их производительность для расчета общего времени взлома.
* **Математический анализ:** Расчет энтропии по Шеннону на основе состава алфавита введенных данных.
* **Безопасная интеграция с внешними базами:** Проверка пароля на компрометацию через API Have I Been Pwned с использованием криптографического хэширования SHA-1 и протокола k-Anonymity (передача только префикса хэша).

## 5. Внешние интеграции и Аналитика
* **Внешние API:** Minerstat API v2.0 (спецификации GPU), Have I Been Pwned API (базы утечек).
* **Аналитический блок:**
  1. Линейный график: Прогнозирование роста устойчивости пароля при увеличении его длины (логарифмическая шкала).
  2. Круговая диаграмма (Doughnut): Визуализация вклада каждой видеокарты в общую мощность вычислительного пула.
  3. Детальная таблица: Распределение хэшрейта в абсолютных (GH/s) и процентных значениях.
