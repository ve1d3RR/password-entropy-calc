# Техническое задание на проект: Калькулятор энтропии паролей для ИБ-специалистов с целью расчета реального времени взлома на современных видеокартах.

## 1. Цель проекта
Создание специализированного веб-сервиса для ИБ-аудиторов и системных администраторов. Проект решает проблему наглядной демонстрации уязвимости коротких паролей. Система математически рассчитывает энтропию введенного пароля, оценивает время его полного перебора (Brute-force) на мощностях современных видеокарт (GPU) и безопасно проверяет факт компрометации пароля в глобальных утечках данных без передачи самого пароля в открытом виде.

## 2. Роли пользователей
* **Гость:** Может воспользоваться быстрым калькулятором на главной странице без сохранения результатов в базу данных.
* **ИБ-специалист (Авторизованный пользователь):** Имеет доступ к личному кабинету. Может создавать проекты (аудируемые компании), добавлять свои конфигурации оборудования (GPU-фермы) и сохранять историю проверок для генерации отчетов.
* **Администратор (Superuser):** Управляет справочником видеокарт и пользователей через стандартную админ-панель Django.

## 3. Модели данных (Сущности)
В проекте реализованы 3 связанные кастомные модели:

* **Модель 1: `AuditProject` (Проект аудита)**
  * Сущность, группирующая проверки для конкретного клиента или отдела.
  * Поля: `user` (ForeignKey на пользователя), `name` (Название проекта/компании), `description` (Описание), `created_at` (Дата создания).

* **Модель 2: `HardwareRig` (Оборудование для взлома)**
  * Профиль вычислительной мощности, имитирующий установку злоумышленника.
  * Поля: `user` (ForeignKey на пользователя - кто добавил конфигурацию), `name` (Название, например "Ферма 4x RTX 4090"), `hashrate_md5` (Скорость перебора MD5 хэшей в секунду), `hashrate_sha256` (Скорость перебора SHA-256).

* **Модель 3: `PasswordAuditLog` (Журнал проверки)**
  * Запись о проведенном тесте. **Важно:** сам пароль в целях безопасности в БД не сохраняется!
  * Поля: `project` (ForeignKey на AuditProject), `hardware` (ForeignKey на HardwareRig), `password_length` (Длина пароля), `entropy_score` (Рассчитанная энтропия в битах), `time_to_crack_seconds` (Расчетное время взлома), `is_pwned` (Boolean - найден ли в утечках), `tested_at` (Время проверки).

## 4. Ключевой функционал (User Stories)
* **Анализ энтропии (Математический модуль):** Пользователь вводит пароль в форму. Система "под капотом" вычисляет размер алфавита (цифры, спецсимволы, регистр) и рассчитывает энтропию по формуле Шеннона.
* **Расчет Brute-Force:** Пользователь выбирает из выпадающего списка видеокарту (HardwareRig). Система делит количество возможных комбинаций пароля на хэшрейт видеокарты и выводит время взлома (от "Мгновенно" до "2 миллиона лет").
* **Безопасная проверка утечек (k-Anonymity):** Система хэширует пароль локально в SHA-1, берет первые 5 символов хэша и отправляет их во внешнее API. Получив ответ, система проверяет совпадения. Пароль не передается по сети в открытом виде.
* **Ведение журнала аудита:** ИБ-специалист сохраняет результаты проверок в рамках выбранного проекта (AuditProject) для формирования статистики защищенности компании.

## 5. Внешние интеграции / Аналитика
* **Интеграция с внешним API:** Использование открытого API **Have I Been Pwned (Pwned Passwords API v3)** для проверки факта компрометации паролей через метод k-Anonymity. Библиотека `requests`.
* **Внутренняя аналитика и визуализация:** Использование библиотеки **Plotly** (или Chart.js на Frontend). В личном кабинете строится график логарифмической зависимости: "Как увеличивается время взлома при добавлении каждого нового символа к паролю на выбранной видеокарте".
